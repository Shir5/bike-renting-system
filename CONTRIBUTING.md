# Участие в разработке

Спасибо за интерес к Bike Renting System. Небольшие исправления можно сразу предлагать pull request; для значительных изменений API, модели данных или архитектуры сначала создайте issue и согласуйте контракт.

## Подготовка окружения

1. Сделайте fork и создайте ветку от актуальной `main`.
2. Запустите инфраструктуру и клиент по корневому `README.md`.
3. Не добавляйте в Git `.env`, ключи, токены, дампы БД и пользовательские данные.

Рекомендуемые имена веток: `feature/short-name`, `fix/short-name`, `docs/short-name`.

## Правила изменений

- Сохраняйте слои backend: controller → service → repository; наружу возвращайте DTO, а не JPA entities.
- Все новые настройки передавайте через переменные окружения с безопасным development default и документируйте в `.env.example` и README.
- Для изменений API сначала обновляйте OpenAPI-контракт, затем генерируйте клиент командой `npm run gen:api`.
- Добавляйте тест, который падает до исправления и проходит после него.
- Не смешивайте в одном pull request рефакторинг и несвязанное функциональное изменение.

## Проверки перед pull request

```bash
cd backend
./mvnw test

cd ../frontend/bike-renting
npm ci
npm test
npm run typecheck
npm run lint
npm run format:check
```

Для изменения нативных функций дополнительно запустите подходящий Detox-сценарий.

## Pull request

В описании укажите проблему, выбранное решение, способ ручной проверки и изменения конфигурации/контракта. Приложите скриншоты для UI-изменений. PR должен быть небольшим, иметь зелёный CI и не содержать сгенерированных или IDE-файлов, не относящихся к задаче.

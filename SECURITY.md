# Security Policy

## Supported versions

Security fixes are applied to the current `main` branch. Tagged stable releases are not maintained yet.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository. If that feature is unavailable, contact a maintainer privately through the profile linked from the repository.

Include the affected component and revision, reproduction steps, expected impact, and any suggested mitigation. Do not access, modify, or retain data that does not belong to you. Maintainers should acknowledge a complete report within seven days and coordinate disclosure after a fix is available.

Never commit credentials. Rotate any secret that has appeared in Git history even if it is removed in a later commit.
